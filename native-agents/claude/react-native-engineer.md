## 2. Architecture

### New Architecture Components
| Component | Purpose | Status |
|-----------|---------|--------|
| **Fabric** | New rendering system (synchronous layout) | Default in RN 0.76+ |
| **TurboModules** | Lazy-loaded native modules | Default in RN 0.76+ |
| **JSI (JavaScript Interface)** | Direct C++ binding, no bridge serialization | Enables synchronous native calls |
| **Hermes** | JavaScript engine optimized for mobile | Default in RN 0.70+ |

### Architecture Comparison
```typescript
// Legacy Bridge (RN < 0.68) — asynchronous, serialized JSON
NativeModules.MyModule.doSomething(data, (result) => {
  console.log(result); // callback after bridge round-trip
});

// New Architecture (JSI + TurboModules) — synchronous, native memory
// MyModule.ts — generated by codegen
import MyModule from 'MyModule';
const result = MyModule.doSomething(data); // direct call
console.log(result);
```

### Interop Strategy
```typescript
// Gradual migration: legacy module wrapped for new arch
import { requireNativeComponent } from 'react-native';

// Fabric-compatible component
const NativeMapView = requireNativeComponent('RNTMapView');

function MapView(props: MapViewProps) {
  return (
    <NativeMapView
      {...props}
      onRegionChange={props.onRegionChange}
    />
  );
}
```

## 5. Performance

### List Virtualization
```typescript
// FlatList — optimized for mobile
<FlatList
  data={items}
  renderItem={({ item }) => <ListItem item={item} />}
  keyExtractor={(item) => item.id}
  // Performance props:
```

## 6. Native Modules

### Swift/Kotlin Interop
```swift
// iOS — Swift module
@objc(MyModule)
class MyModule: NSObject {
  @objc
  func doSomething(_ data: String, resolver resolve: @escaping RCTPromiseResolveBlock,
                   rejecter reject: @escaping RCTPromiseRejectBlock) {
    // Native implementation
    resolve(result)
  }
}
```

```kotlin
// Android — Kotlin module
class MyModule(reactContext: ReactApplicationContext) :
  ReactContextBaseJavaModule(reactContext) {

  override fun getName(): String = "MyModule"

  @ReactMethod
  fun doSomething(data: String, promise: Promise) {
    // Native implementation
    promise.resolve(result)
  }
}
```

### Expo Modules

## 9. Common Anti-Patterns

| Pattern | Why | Action |
|---------|-----|--------|
| Ignoring platform guidelines | Users feel the app is "web-wrapped" | Follow Apple HIG and Material Design separately |
| Heavy JS thread | UI jank from expensive computations | Offload to worklets (react-native-reanimated) or native |
| Overusing `any` in TypeScript | Native module types lost | Use `codegen` for typed native modules |
| No error boundaries in native modules | Crash across entire app | Wrap native components in try-catch |
| FlatList without optimization | Laggy scrolling on large lists | Use FlashList or add FlatList performance props |
| Inline styles everywhere | No performance benefit in RN | StyleSheet.create() or external styles |
| Ignoring hermes | Slower startup, larger bundle | Enable Hermes bytecode |
