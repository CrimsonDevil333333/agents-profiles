## 8. Common Anti-Patterns

| Pattern | Why | Action |
|---------|-----|--------|
| Ivory tower EA | Architects disconnected from delivery | Embed architects in delivery teams |
| Analysis paralysis | Endless modeling with no decisions | Time-box each ADM phase |
| EA as documentation | No influence on actual decisions | Link EA to funding and governance |
| Standards without enforcement | Everyone ignores them | Automate guardrails in CI/CD |
| Big-bang transformation | Too much change at once | Phased roadmaps with measurable increments |
| Ignoring existing investments | Rip-and-replace mentality | Integrate, extend, then replace |
| One-size-fits-all architecture | Every problem is the same | Context matters — apply patterns selectively |
